PHASE 2 — SYNC PIPELINE API (~1:24 AM, Jul 12, 2026)
--------------------------------------------------------------------------------

USER ASKED:
  Build a sync pipeline that doesn't lie or duplicate data — CRM, payments,
  calendar sources with incremental + full fetch, idempotent writes, fault
  isolation. Use Express.js, mock data, SOLID principles, and Adapter pattern.

WHAT WAS BUILT:

  Architecture:
    src/
    ├── adapters/       # CRM (HubSpot), payments (Stripe), calendar (Google)
    ├── ports/          # ISourceAdapter, IRecordRepository interfaces
    ├── domain/         # Normalized schema + sync results
    ├── mock/           # Seeded mock stores (5 records each)
    ├── services/       # Orchestrator, per-source sync, webhooks, cursors
    ├── repositories/   # Idempotent upsert store
    ├── api/            # Express routes
    └── container.js    # DI composition root

  Key behaviors implemented:

    Requirement                    | Solution
    ---------------------------------|------------------------------------------
    Different shapes → one schema    | Adapters + normalizers map vendor fields
    Incremental + full fetch         | fetchIncremental(cursor) + fetchFull()
    Stale cursor / 410 fallback      | Auto full backfill on StaleCursorError /
                                     | ExpiredTokenError
    Idempotent writes                | Upsert by {source}:{externalId} + contentHash
    Fault isolation                  | Sources run in parallel; one failure
                                     | doesn't block others
    Garbage tolerance                | Invalid records skipped with warning

  API endpoints:

    Method | Path                    | Purpose
    -------|-------------------------|------------------------------------------
    POST   | /api/sync               | Sync all 3 sources
    POST   | /api/sync/:source       | Sync one source
    GET    | /api/records            | List normalized records
    POST   | /api/webhooks/:source   | Idempotent webhook ingestion
    GET    | /api/sync/status        | Last run + cursors

  Verified:
    - 15 records ingested (5 per source)
    - Re-run produced 0 duplicates
    - Stale cursor triggered backfill
    - Payments-down scenario still synced CRM + calendar

  summary + breakdown views that always agree, Strategy Pattern, Supabase Postgres
  storage, finance app test-mode sample data.

WHAT WAS BUILT:

  Strategy Pattern architecture:
    src/metrics/
    ├── strategies/
    │   ├── revenue-calculation-strategy.js   # Interface
    │   └── allow-list-collected-strategy.js  # Single source of truth
    ├── revenue-definition-registry.js        # Throws if 2nd strategy registered
    ├── revenue-consistency-guard.js          # Summary must equal sum(breakdown)
